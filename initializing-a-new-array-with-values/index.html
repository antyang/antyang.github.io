Initializing a new Array with values

March 31, 2017

This is one of many ways to initialize a new array, specifically using new Array(n).

Say we want to initialize a new Array with a short length of 3.

We can do: 

var shortArr = [null, null, null];

Then iterate over the array and fill it. 

shortArr = arr.map(function(el, i) {  
    return i;
}
// => [0, 1, 2]

Great!


Now what if we want to initialize a longer array, say 13 or greater?

We can do the same…
var longArr = [null, null, null, null, null, null, null, null, null, null, null, null, null];

No - Don't be silly.


One way to do this is to create a new empty array with:
new Array(arrayLength) and iterate with map() resulting in a new array. caveat - map() ignores and preserve "holes" resulting in an empty array with "holes".

new Array(13).map(function(el, i) {  
    return i;
})
// => [ , , , ...];

A way to patch these missing elements is to use Function.prototype.apply()which treats holes as if they were undefined elements. 

var longArr = Array.apply(null, Array(13));  
// => [undefined, undefined, undefined...]
longArr.map(function(el, i) {  
    return i;
})
// => [0, 1, 2, 3, 4, 5, ...]
