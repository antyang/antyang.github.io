Setting up Ghost with Github pages

  • _
  • email

Ghost is a "blogging platform" that's known for being user friendly and beautifully designed straight out of the box. Github Pages is a static site host on Github for hosting simple blogs and landing pages for open-source projects.

Starting out

Create a folder to hold Ghost and static files. 

$ cd <to_desired_location>
$ mkdir <ghost_folder_name> && cd <ghost_folder_name>

Make sure you have Node and Python (default on apple macs).

Installing homebrew

homebrew (OSX package manager) allows you to install, remove and updated applications and packages.

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

Installing wget

wget is a tool used to retrieve content from web servers. With homebrew installed, we can run the following.

$ brew update
$ brew install wget

Installing Buster and Pip

Buster allows you to export content into static pages. Here we'll install Buster with pip (python package manager)

// go to ghost's directory
$ cd <ghost_folder_name>

// if you don't have pip installed
$ sudo easy_install pip

// install buster
$ sudo pip install buster

Installing Ghost and start your local Ghost server

// gets the latest version of ghost
$ wget https://ghost.org/zip/ghost-latest.zip

// unzips ghost
$ unzip ghost-latest.zip

// installs dependencies
$ npm install

$ npm start

$ open http://localhost:2368/ghost

npm start starts the Node.js server that will serve up the admin interface at http://localhost:2368/ghost. Here is where you will create an account and manage all your contents and fancy settings.

Coming together

On github create a new repo [your-github-username].github.io.

NOTE: must be specific format. Otherwise it will not work.

// current terminal tab is running so...
// (in a new tab) get back to the ghost folder
$ cd <ghost_folder_name>

$ buster setup --gh-repo=\
"https://github.com/<username>/<username>.github.io.git"

$ buster generate

$ buster deploy

buster generate will create all of the appropriate directories, HTML pages and links for your entire Ghost blog within that folder.

buster deploy, adds, commits and pushes the changes you generated up to your blog's repo.

To exit hit Ctrl-C in the tab running Ghost's server.

recap

$ cd <ghost_folder_name>
$ npm start
$ buster generate
$ buster deploy

enjoy.
